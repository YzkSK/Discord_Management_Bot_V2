# i18n設計：Botとダッシュボードの多言語対応

**日付:** 2026-05-27  
**ステータス:** 承認済み  
**スコープ:** GuildごとのBot返答・ログメッセージの言語切り替え（英語 / 日本語）。設定はWebダッシュボードで実施。

---

## 概要

Discordボットのログメッセージ・スラッシュコマンド応答を英語と日本語で切り替えられるようにする。Guild管理者がダッシュボードの設定ページから言語を選択し、DBに保存する。Botはイベント処理・コマンド応答時にDBから言語設定を読み込み、対応するロケールで文言を組み立てる。

---

## アーキテクチャ

### アプローチ

`packages/shared` に型付きロケールオブジェクトを用意する。i18nライブラリは使わず、TypeScriptのオブジェクトと関数で実現する。型安全・追加依存なし・既存パターンと一貫性がある。

### データフロー

```
ダッシュボード（設定ページ）
  → PATCH /api/settings { language }
  → updateGuildConfigByGuildId(db, { guildId, language })
  → guild_configs.language が PostgreSQL に保存される

Bot（イベント処理・コマンド応答）
  → getGuildConfigByGuildId(db, guildId)
  → getLocale(config.language ?? 'en')
  → loc.someKey / loc.someKey({ var }) → 言語に応じた文字列
  → Discordにメッセージ送信
```

## Section 1: データベース

**ファイル:** `packages/db/src/schema/core.ts`

`guildConfigs` テーブルに `language` カラムを追加する：
- 型: `text`
- デフォルト: `'en'`
- CHECK制約: `language IN ('en', 'ja')`

```typescript
language: text("language").notNull().default("en")
// check: guild_configs_language_check → language IN ('en', 'ja')
```

**ファイル:** `packages/db/src/repositories/guilds.ts`

- `getGuildConfigByGuildId` のSELECTに `language` を追加する
- `updateGuildConfigByGuildId` の引数に `language?: string` を追加する（型バリデーションはAPI境界で実施するため `string` 型）

**マイグレーション:** `packages/db/drizzle/` に新しいDrizzleマイグレーションファイルを生成する

## Section 3: Bot の変更

### `apps/bot/src/discord/log-channel.ts`

- `formatLogEventTitle(eventName, loc)` → `loc.logTitle({ eventName })` を使う
- `formatLogEventLines(event, loc)` → `loc.logActor`・`loc.logChannel` などを使う
- `sendEventToConfiguredLogChannel(client, event, db)` に `db: DbClient` 引数を追加：
  1. `getGuildConfigByGuildId(db, event.guildId)` でguild設定を取得
  2. `const loc = getLocale(config?.language ?? 'en')`
  3. `loc` をフォーマット関数に渡す

### `apps/bot/src/discord/log-writer.ts`

- `options.db` を `sendEventToConfiguredLogChannel` に渡す（`CreateDiscordLogWriterOptions` はすでに `db` を持っているため変更不要）

### `apps/bot/src/commands/setup.ts`

各サブコマンドハンドラーの冒頭で言語を解決する：

```typescript
const config = await getGuildConfigByGuildId(context.db, guildId).catch(() => null);
const loc = getLocale(config?.language ?? 'en');
```

ハードコードされた英語文字列をすべて `loc.*` に置き換える。

### `apps/bot/src/commands/recruitment.ts`

同じ言語解決パターンを適用。返答文字列をすべて `loc.*` に置き換える。

## Section 4: ダッシュボードの変更

### `apps/dashboard/src/app/api/settings/route.ts`

**GET:** レスポンスJSONに `language: config.language` を追加する。

**PATCH:**
- リクエストボディから `language` を取り出す
- `isGuildLanguage(language)`（`@discord-bot/shared` からインポート）でバリデーション。不正な値は400を返す
- `updateGuildConfigByGuildId` に `language` を渡す

### `apps/dashboard/src/app/settings/settings-panel.tsx`

- `SettingsResponse` インターフェースに `language: string` を追加する
- `const [language, setLanguage] = useState('en')` を追加する
- `loadSettings`: 取得後に `setLanguage(data.language)` を呼ぶ
- `saveSettings`: PATCHリクエストのボディに `language` を含める
- UI: Log Modeセレクトの下にLanguageセレクトを追加する：

```tsx
const languageOptions = [
  { label: "English", value: "en" },
  { label: "日本語", value: "ja" }
];
```

## フォールバック戦略

| 状況 | フォールバック |
|------|--------------|
| guildIdなし（DM等） | `'en'` |
| GuildがDBに未登録 | `'en'` |
| DB取得エラー | `'en'` |
| DB内に不明な言語値 | `'en'`（`isGuildLanguage` ガードで検出） |

- Discordコマンドピッカーに表示されるスラッシュコマンドの `description` — 英語のみ
- ユーザーごとの言語設定
- 3言語以上への拡張
