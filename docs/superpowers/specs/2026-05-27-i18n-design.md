# i18n Design: Bot & Dashboard Multilingual Support

**Date:** 2026-05-27  
**Status:** Approved  
**Scope:** Per-guild language selection (English / Japanese) for Discord bot messages and dashboard settings UI

---

## Overview

Add per-guild language support to the Discord bot and Web Dashboard. Guild admins select English or Japanese via the Dashboard settings page. The bot uses the stored language when formatting Discord log channel messages and slash command responses.

---

## Architecture

### Approach

Typed locale objects in `packages/shared`. No i18n library — plain TypeScript objects with string values and typed functions for variable interpolation. Fully type-safe, zero additional dependencies, consistent with existing patterns.

### Data Flow

```
Dashboard (settings page)
  → PATCH /api/settings { language }
  → updateGuildConfigByGuildId(db, { guildId, language })
  → guild_configs.language stored in PostgreSQL

Bot (event/command handling)
  → getGuildConfigByGuildId(db, guildId)
  → getLocale(config.language ?? 'en')
  → loc.someKey / loc.someKey({ var }) → localized string
  → Discord message
```

## Section 1: Database

**File:** `packages/db/src/schema/core.ts`

Add `language` column to `guildConfigs` table:
- Type: `text`
- Default: `'en'`
- CHECK constraint: `language IN ('en', 'ja')`

```typescript
language: text("language").notNull().default("en")
// check: guild_configs_language_check → language IN ('en', 'ja')
```

**File:** `packages/db/src/repositories/guilds.ts`

- `getGuildConfigByGuildId`: include `language` in SELECT
- `updateGuildConfigByGuildId`: add optional `language?: string` to input; include in upsert SET when provided (type is `string` since validation occurs at the API boundary before this function is called)

**Migration:** New Drizzle migration file in `packages/db/drizzle/`

## Section 3: Bot Changes

### `apps/bot/src/discord/log-channel.ts`

- `formatLogEventTitle(eventName: string, loc: Locale)` → uses `loc.logTitle({ eventName })`
- `formatLogEventLines(event: NormalizedEvent, loc: Locale)` → uses `loc.logActor`, `loc.logChannel`, etc.
- `sendEventToConfiguredLogChannel(client, event, db)` — add `db: DbClient` parameter:
  1. Fetch `getGuildConfigByGuildId(db, event.guildId)`
  2. `const loc = getLocale(config?.language ?? 'en')`
  3. Pass `loc` to formatting functions

### `apps/bot/src/discord/log-writer.ts`

- Pass `options.db` to `sendEventToConfiguredLogChannel` — no interface changes needed since `db` is already in `CreateDiscordLogWriterOptions`

### `apps/bot/src/commands/setup.ts`

Language resolution pattern at the top of each subcommand handler:
```typescript
const config = await getGuildConfigByGuildId(context.db, guildId).catch(() => null);
const loc = getLocale(config?.language ?? 'en');
```

Fallback `'en'` used when:
- No `guildId` (DM context — pre-guild error messages)
- `getGuildConfigByGuildId` returns `null` (guild not yet set up)

All hardcoded English strings replaced with `loc.*` calls.

### `apps/bot/src/commands/recruitment.ts`

Same language resolution pattern. All reply strings replaced with `loc.*`.

## Section 4: Dashboard Changes

### `apps/dashboard/src/app/api/settings/route.ts`

**GET:** Add `language: config.language` to response JSON.

**PATCH:** 
- Extract `language` from request body
- Validate with `isGuildLanguage(language)` (imported from `@discord-bot/shared`); return 400 if invalid
- Pass `language` to `updateGuildConfigByGuildId`

### `apps/dashboard/src/app/settings/settings-panel.tsx`

- `SettingsResponse` interface: add `language: string`
- Add `language` state: `const [language, setLanguage] = useState('en')`
- `loadSettings`: set `setLanguage(data.language)` after load
- `saveSettings`: include `language` in PATCH body
- UI: add Language `<Select>` below the Log Mode selector:

```tsx
const languageOptions = [
  { label: "English", value: "en" },
  { label: "日本語", value: "ja" }
];
```

---

## Fallback Strategy

| Situation | Fallback |
|-----------|----------|
| No guildId (DM) | `'en'` |
| Guild not set up in DB | `'en'` |
| DB error on language fetch | `'en'` |
| Unknown language value in DB | `'en'` (via `isGuildLanguage` guard) |

---

## Out of Scope

- Dashboard UI strings (labels, buttons, error messages in the Next.js app) — English only
- Slash command `description` fields shown in Discord's command picker — English only
- Per-user language preferences
- More than 2 languages
