# Temp VC

## 動作仕様

- 設定済みの作成VCが起動トリガーになる。
- Bot 以外のメンバーが作成VCに参加すると、Bot が `🎮 {ユーザー名}` という名前の生成VCを作成する。
- メンバーは生成VCに自動移動される。
- 生成VCの隣に `control-🎮 {ユーザー名}` というプライベートコントロールテキストチャンネルが作成される。
- コントロールチャンネルは Temp VC オーナーにのみ表示される。
- コントロールチャンネルにはオーナー情報とチャンネル情報を含む Components V2 コントロールメッセージが投稿される。
- コントロール操作は現在の Temp VC オーナーのみが使用できる。

### コントロール操作

- **リネーム** — 生成VCの名前を変更する
- **ロック / アンロック** — 参加権限をトグルする
- **非表示 / 表示** — チャンネルの可視性をトグルする
- **ユーザー上限** — メンバー上限を設定する（0〜99）
- **ビットレート** — チャンネルのビットレートを設定する（8〜384 kbps）
- **キック** — 選択したメンバーを Temp VC から切断する

### 所有権の移転

- オーナーが Temp VC を退出すると、参加順序が次のアクティブメンバーに所有権が移る。
- オーナーが 10 分間非アクティブな場合も所有権が移転する。
- 所有権変更時にプライベートコントロールチャンネルの権限が新オーナーに移動する。
- `voice.temp.owner_transferred` としてロギングされる。

### 削除

- Temp VC が空になると、5 秒後に削除がスケジュールされる。
- 生成VCが削除されると、コントロールテキストチャンネルも削除される。
- `voice.temp.deleted` としてロギングされる。

### ロギング

- `voice.temp.created` — Temp VC 作成時。ペイロードに `callSessionId`、`ownerId`、チャンネル名を含む
- `voice.temp.owner_transferred` — 所有権変更時
- `voice.temp.deleted` — 削除時
- 内部的なチャンネル作成・削除イベントは一般ログから抑制される

## データベース

| テーブル | 用途 |
|---|---|
| `tempVoiceChannels` | 生成チャンネルの追跡（`guildId`, `channelId`, `ownerId`, `creationChannelId`, `controlChannelId`, `callSessionId`, `deleteScheduledAt`） |
| `callSessions` | ボイスセッション状態（`guildId`, `channelId`, `status`, `statusMessageId`, `startedAt`, `endedAt`） |
| `callSessionMembers` | セッション参加者（`callSessionId`, `userId`, `joinOrder`, `joinedAt`, `leftAt`） |

`guildConfigs` フィールド:

- `tempVoiceCreateChannelId` — トリガーとなるVCのチャンネルID
- `tempVoiceCategoryId` — 生成チャンネルを配置するカテゴリ（任意）

## セットアップ

マイグレーションを適用する。

```bash
pnpm db:migrate
```

スラッシュコマンドを登録する（必要な場合）。

```bash
pnpm --filter @discord-bot/bot commands:register
```

Discord でセットアップコマンドを実行する。

```text
/setup temp-vc creation-channel:<音声チャンネル> category:<カテゴリ（任意）>
```

オプション:

- `creation-channel`（必須）— Temp VC 作成のトリガーとなる音声チャンネル
- `category`（任意）— 生成チャンネルを配置するカテゴリチャンネル

## Bot に必要な権限

- チャンネルを見る
- チャンネルの管理（ロック/アンロック、非表示/表示、リネーム、ユーザー上限、ビットレート、および `/setup temp-vc` に必要）
- メンバーの移動
- メッセージの送信
- メッセージ履歴の閲覧
- 音声チャンネルへの接続
- 監査ログの表示（任意、抑制されたチャンネルイベントの情報付与に使用）

## 検証

### コントロールチャンネル

1. 設定した作成VCに参加する。
2. `🎮 {ユーザー名}` という音声チャンネルが作成されることを確認する。
3. 生成VCに自動移動されることを確認する。
4. Components V2 メッセージを含むプライベートコントロールチャンネルが作成されることを確認する。
5. **リネーム**を押して新しい名前を入力し、VCの名前が変わることを確認する。
6. **ロック**を押して新規参加できなくなることを確認する。
7. **アンロック**を押して参加できるようになることを確認する。
8. **非表示**を押してチャンネルが隠れることを確認する。
9. **表示**を押してチャンネルが見えるようになることを確認する。
10. **ユーザー上限**を設定してチャンネル上限が変わることを確認する。
11. **ビットレート**を設定してビットレートが変わることを確認する。
12. キックセレクターでメンバーを選択して切断されることを確認する。
13. オーナー以外がコントロール操作を試みて拒否されることを確認する。

### 所有権の移転

Bot 以外の 3 ユーザー以上で実施する。

1. ユーザー A が作成VCに参加してオーナーになる。
2. ユーザー B が生成 Temp VC に参加する。
3. ユーザー C が B の後に参加する。
4. コントロールチャンネルがユーザー A にのみ表示されることを確認する。
5. ユーザー A が退出する。
6. 参加順序が早いユーザー B がオーナーになることを確認する。
7. コントロールチャンネルの権限がユーザー B に移ることを確認する。
8. ユーザー B がコントロールを操作できることを確認する（ユーザー C はできない）。
9. ログに `voice.temp.owner_transferred` が記録されることを確認する。

### クリーンアップ

10. 生成VCを退出する。
11. 約 5 秒後に両方の生成チャンネルが削除されることを確認する。
