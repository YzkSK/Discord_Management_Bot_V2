# Dashboard ページ一覧

Dashboard は Next.js アプリで、ローカル開発では `http://localhost:3000`、本番では設定した `PUBLIC_DASHBOARD_URL` で利用できる。

全ページで認証が必要。未認証ユーザーは `/login` にリダイレクトされる。

## ページ一覧

### `/` — 概要

ギルド選択と機能ページへのクイックリンク。選択したギルドの設定済み機能と Bot ステータスの概要を表示する。

### `/logs` — ログビューアー

ゲートウェイイベントログの閲覧と検索。

機能:

- **カテゴリタブ**: すべて、メッセージ、ボイス、Temp VC、募集、監査、TTS、システム
- **リアルタイム状態**: idle / connecting / live / offline / error
- **Human View**: 読みやすいイベントサマリー（全ロールで閲覧可能）
- **Raw JSON**: フルイベントペイロード（`owner` と `admin` のみ）
- **フィルター**: イベント名、アクターID、チャンネルID、メッセージID、フリーテキスト検索
- **ページネーション**: カーソルベース

### `/settings` — ギルド設定

機能セクション別に整理された設定ページ。

| セクション | 内容 | 必要なロール |
|---|---|---|
| 概要 | ギルドID、ギルド名、アクセスロール、機能の設定状況 | `viewer` |
| ログ | ログモード、Dashboard 言語 | `admin` |
| Temp VC | 作成チャンネルID、カテゴリID | `admin` |
| TTS | TTS テキストチャンネルID | `admin` |
| 募集 | 募集チャンネルID | `admin` |
| Dashboard アクセス | ユーザー/ロールグラント、管理ロールID | `owner` |

### `/voice` — ボイスアクティビティ

- 参加者数とチャンネル名を含むアクティブなボイスセッション
- 最近（終了済み）のセッション
- Temp VC チャンネルの状態（オープン中のチャンネル、オーナー、作成チャンネル）
- `/setup voice-status` と `/setup temp-vc` へのセットアップショートカット

### `/recruitment` — 募集管理

- 募集投稿のステータス（open / full / closed の件数）
- 各投稿の参加者数と作成者情報
- 募集イベントのログへのリンク

### `/tts` — TTS 設定

- 設定済み TTS テキストチャンネル
- ギルドデフォルト VOICEVOX スピーカー
- ユーザー別スピーカー設定
- 辞書エントリ数と表示可能なエントリ一覧

### `/health` — システムヘルス

- 全体ステータス: `OK` またはエラー
- 依存サービス行: PostgreSQL、Redis、VOICEVOX — 各ステータスとレイテンシ
- 手動リフレッシュボタン
- いずれかの依存サービスが利用できない場合は HTTP `503` を返す

## ロール別アクセス制御

| 機能 | `viewer` | `admin` | `owner` |
|---|---|---|---|
| 全ページを閲覧 | ✓ | ✓ | ✓ |
| ログの Human View | ✓ | ✓ | ✓ |
| ログの Raw JSON | — | ✓ | ✓ |
| 設定の編集 | — | ✓ | ✓ |
| Dashboard グラント管理 | — | — | ✓ |

## 検証

アプリスタックを起動する。

```bash
docker compose --profile app up -d --build bot dashboard
```

`http://localhost:3000` を開く。最初のパスにはサーバーオーナーのアカウントを使用し、その後 `admin` と `viewer` グラントでも確認する。

### ログ

- カテゴリタブが表示され、イベントが正しくフィルターされることを確認する。
- ソケット接続時にリアルタイム状態が変わることを確認する。
- `owner` / `admin` は Raw JSON が表示され、`viewer` では非表示であることを確認する。

### 設定

- 概要に機能の設定状況が表示されることを確認する。
- `admin` としてログ設定（ログモード、言語）を保存する。
- `admin` として Temp VC チャンネルIDを保存する。
- `admin` として TTS テキストチャンネルIDを保存する。
- `owner` としてユーザーまたはロールの Dashboard グラントを追加・編集・削除する。

### ボイス

- ユーザーが音声チャンネルにいるときにアクティブな通話が表示されることを確認する。
- 通話終了後も最近のセッションが残ることを確認する。
- 生成された Temp VC が存在するときに Temp VC 行が表示されることを確認する。

### 募集

- `/recruitment create` 後に募集投稿が表示されることを確認する。
- open / full / closed の件数が表示行と一致することを確認する。
- Join と Leave 操作後に参加者数が更新されることを確認する。

### TTS

- `/setup tts` 後に設定済みステータスが変わることを確認する。
- admin がスピーカーを設定後にギルドデフォルトスピーカーが表示されることを確認する。
- `/speaker set` 後にユーザー別スピーカー設定が表示されることを確認する。
- 辞書エントリ数が設定内容と一致することを確認する。

### ヘルス

- 全依存サービスが到達可能なときに全体ステータスが `OK` であることを確認する。
- 各依存サービス行にステータスとレイテンシが表示されることを確認する。
